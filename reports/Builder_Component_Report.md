# Builder Component Report

* **Features**: Defines draggable visual component items.
* **Component Types**: Card, Button, Input, Form, Navigation, Tabs, Chart, List, Sidebar, AIWidget, FitnessCard, etc.
* **Metadata Constraints**: Every component holds a permanent, unique Component ID (e.g. `btn_login`, `card_profile`) to allow coding agents to bind properties to future production source files.
