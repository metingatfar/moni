# Visual Hierarchy Report

* **Mechanics**: Governed by `VisualHierarchyEngine` mapping parent-child layout nodes.
* **Component Nesting**: Audits coordinates to determine if elements are contained within Cards, Forms, Modals, or Viewports.
* **Accessibility Audits**: Inspects text sizes, validates that headings are ordered correctly (e.g. without skipping levels), and checks button contrast/labels.
